CP1406 - Web Design & Development - JCU

Practical - Publishing

Aims:

  • To learn to connect to a Web server and upload files for the Web.
  • To be prepared for submitting your assignments. This is super important. Learn this, and don't leave it to the last minute.

First:

  • When using Dreamweaver, follow rule #1 - work in a defined site.
  • Make sure you name your files and directories correctly.
    On our Web server (a Linux machine) and most Web servers in the world, file names are case-sensitive and should not have spaces.
    Your home page file must be called index.html
    Index.html is not index.html is not index.htm is not whatever.whocares...
  • Make sure your site uses relative links (e.g. src="images/file.jpg" not src="file:///C/Users/...").
    (It will have done this by default if you always worked in a defined site.)

Off-Campus Access

You will need to setup and use a VPN (Virtual Private Network) in order to access these servers from outside the JCU network.

Details for installing and setting up GlobalProtect to do this are found here.

Get Connected:

You can use any FTP program that supports SFTP (Secure File Transfer Protocol) including Fetch, Cyberduck, WinSCP or... FileZilla (which is installed on the JCU computers).

  • Run FileZilla. (For your own computer, get the client from https://filezilla-project.org)

We need to connect to the remote Web server using SFTP. The server name is:

  • ditwebcns.jcu.edu.au - for Cairns students
  • ditwebtsv.jcu.edu.au - for Townsville and external students

If you are on your own computer, use the Site Manager (first button) to setup your site so you only have to type it once.

If you are on a lab computer, the easiest way is to enter the details in the Quickconnect section with your server, username and password (your password is the usual one for email, LearnJCU, etc.) and the port number 22, then click the big ol' button.

If you get a message about an unknown host key, just accept it - it's safe :)

You should now have two panes - your local files (left) and the remote server files. You can drag and drop files between panes or to/from Windows Explorer or Finder (Mac).
Yours will look a bit different, but it's something like:

Anything you want to publish on the Web needs to go inside a special folder called:

public_html

To start with, you will not have anything in your public_html folder on the server.
We will be using directories to separate our different sites - so create a directory for any site you want to upload... inside public_html:

  • Create a directory called me
  • Drag the files for your first About Me site into this folder.
    (Don't drag just one folder, drag everything inside the folder, making sure that includes index.html.)

Now to view it online, in a Web browser, you go to :

http://ditwebtsv.jcu.edu.au/~jc123456/me

but change:

  • jc123456 to your own login
  • tsv to cns if you're a Cairns student
  • me to whatever other site folder you create in future

What happened?

  • If this shows you your website, it means you uploaded it correctly (and you have an index.html in the test directory).
  • If you see a list of files, it means your folder has no index.html in it.
  • If you see a link to a folder (e.g. AboutMe), it means you copied in an extra subfolder - delete this from FileZilla (right pane) and upload everything inside the folder, but not the folder itself.
  • If this did not work (404 maybe), please check back through the instructions to see what you missed (usually spelling/typing or where you put the files... remember it must be inside public_html).

You can also upload directly from your editing program if you prefer...

If you're using Dreamweaver:

In Dreamweaver, choose Site > Manage Sites...

  • With your site selected, click on the pencil icon to edit that site.
  • Click in the left menu where it says Servers...
    Click the + button to add a new server.
  • In the dialog box that comes up, enter your details like the image below (ditwebtsv.jcu.edu.au in Tsv or ditwebcns.jcu.edu.au in Cns).

    Make sure you change FTP to SFTP.

  • Click the Test button to make sure Dreamweaver can connect to the server.
    If it can't, check the details carefully and try again.
  • Click Save.

Important: You need to match the root directory of your local site with the "Root Directory" field in the server setup. (
E.g. in the screenshot above, the server root directory is public_html/me - and the site root in Dreamweaver will be a folder containing the index.html file for the "me" site.

You can now upload files and folders by clicking on them in the Files panel and then clicking the up arrow button.

If it asks if you want to put dependent files, choose Yes and it will also upload any images or CSS files that your file references (but not linked html pages).

Upload your pages, then view your site in a Web browser.

Get Familiar:

  • Every time you make a change to your site (locally), you can just upload it over the top of the old site on the Web (remote). That's how you update a site online.
  • Don't let this be the last time you upload to the server before your assignment is due. Try a few sites (with different folder names, make some changes and re-upload them...
    Make sure you know how to do this.


Tech Note: The following should be done for you automatically, but if not...
If you get 403 Forbidden for folders and you do want the browser to show the contents of directories that do not have an index.html/php file in them, you can create a file called .htaccess inside public_html that simply contains the following line:
Options +Indexes
